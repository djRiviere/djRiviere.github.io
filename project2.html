PROJECT 1: Mastermind

Concept

Mastermind

This game is about a 4 length color code which is unknown to the player. The player has to then guess the code and the computer will tell them if they have colors in the correct position or in the incorrect position. I came up with this game because I used to play it with my father when I was a kid. And it was a pretty interesting idea and a good start for one of my first projects.

How i did it

I started thinking of ways to let the computer check whether a color was in the right spot or if the color was even in the sequence. And although this was my first time of really learning how to use lists. I decided to save variables in lists and compare those indexes to the indexes in the answer(the list generated by the computer). If indexes matched I added 1 to a variable named (right spot) This would let the player know if he was right by printing a statement with the number of right colors in the right spot. For the colors in the wrong spot it would check if there were simulair colors in those two lists minus the ones in the right spot to sign which ones were the right color but not in the right spot. This was done with two alternative lists that wouldn’t get hurt by having numbers subtracted off of them. These were the most fundamental princibles of this code

What i learned

This was the first time I coded a game. It was all new and I didn’t really know where to start. Eventually I learned how to use lists and it became allot easier. I previously made menus in python and that’s how I started out. And this was a big step in the right direction. I learned how to use the tools I knew how to use to the best extent and got a lot more familiar with loops, If statements, While statements, Functions, and (logical or/and operators. I started with python cause it was one of the more beginner friendly languages.
